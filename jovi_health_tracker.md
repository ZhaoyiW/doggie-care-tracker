请帮我开发一个移动端优先的中文 Web App，名字叫「狗狗健康记录」。

产品目标：
这是一个用于长期记录狗狗日常生活和健康状况的工具，重点是记录饮食、遛狗、排便情况、疫苗、就医记录和健康检查，并能通过日历和趋势帮助我判断健康问题（比如腹泻的诱因）。

目标用户：
单个宠物主人，主要在手机上使用，需要快速记录，操作简单。

设计要求：
- 中文界面
- 移动端优先（mobile-first）
- 风格温和、干净、略带可爱但不过度
- 卡片式布局
- 输入尽量简单，减少打字
- 使用适量 emoji（🍚🐕💩💉🏥🧪）

时间输入要求：
- 所有时间字段使用“半小时粒度”
- 提供固定选项（00:00, 00:30, 01:00...23:30）
- 默认选中当前时间（向下取整到最近半小时）

核心功能：

1）饮食记录
字段包括：
- 日期
- 时间（半小时选择）
- 食物类型（狗粮 / 处方粮 / 鸡肉米饭 / 零食 / 其他）
- 食物内容（自由输入）
- 数量
- 是否吃完
- 备注

2）遛狗 + 排便记录
每次遛狗记录：
- 日期
- 时间
- 备注

每次遛狗可以记录多个排便事件（Poop Event）：

排便事件字段：
- 时间（半小时选择）
- 状态：
  - 正常
  - 软便
  - 拉稀
  - 蹲了但没拉出来
- 是否有黏液（是/否）
- 是否有血（是/否）
- 是否只有滴水（是/否）
- 备注

需要提供一个“快速记录排便”的方式：
用户可以快速选择状态并保存，减少输入步骤。

3）每日状态记录（可选）
- 精神：正常 / 稍差 / 很差
- 食欲：正常 / 一般 / 差
- 是否呕吐：无 / 有
- 饮水：正常 / 偏多 / 偏少
- 体重（可选）
- 备注

4）日历页面（自动生成标签）

根据当天所有排便数据自动生成标签，规则如下：

优先级从高到低：

- 如果当天有任意一次“拉稀” → 标记为「拉稀」
- 否则如果排便次数 < 2，或存在“蹲了但没拉出来” → 标记为「便秘倾向」
- 否则如果排便次数 ≥ 2 且全部为“正常”或“软便” → 标记为「正常」
- 否则 → 无记录

颜色建议，采用莫兰迪配色：
- 正常：绿色
- 软便：黄色
- 拉稀：红色
- 便秘倾向：橙色
- 无记录：灰色

点击日历某一天可以查看当天所有记录。

5）疫苗记录
- 疫苗名称
- 接种日期
- 下次 due 日期
- 医院
- 备注

首页显示即将到期提醒。

6）就医记录
- 日期
- 医院 / 医生
- 就诊原因
- 诊断
- 治疗方案
- 药物
- 总结
- 是否需要复诊
- 复诊时间

7）健康检查记录
- 日期
- 类型（血检 / 便检 / 尿检 / X光 / 超声 / 其他）
- 结果摘要
- 是否正常
- 详细说明

页面结构：
- 首页 Dashboard
- 每日记录
- 日历
- 医疗记录
- 趋势分析

Dashboard 显示：
- 今天是否已记录
- 最近一次排便情况
- 最近30天拉稀天数
- 下次疫苗提醒
- 最近一次就医记录
- 快捷新增按钮

趋势页面：
- 最近30天拉稀次数
- 正常天数
- 简单趋势图

技术要求：
- 使用现代前端框架（如 React）
- mobile-first 响应式设计
- MVP 使用本地存储（localStorage 或 IndexedDB）
- 支持编辑记录
- 表单简单、快速
- 提供示例数据（Jovi）

数据结构建议：
- DogProfile
- FoodLog
- WalkLog
- PoopEvent
- SymptomLog
- VaccineRecord
- VetVisit
- HealthTest

请先实现一个完整可用的 MVP 版本，重点保证：
- 记录流程顺畅
- 日历标签逻辑正确
- 移动端体验良好

后续再考虑扩展功能（提醒、导出、多宠物等）。
